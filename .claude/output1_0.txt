How to run
1. Install dependencies (from the project root in a terminal with Python):


pip install -r requirements.txt
2. Start the server:


uvicorn backend.main:app --reload --port 8000
3. Open interactive docs:

http://localhost:8000/docs — full schema visible in the Schemas section, try the /parse endpoint directly from the browser.

4. Run the test:


python tests/test_parse.py
Expected output: 4 instruments, ≥56 notes, 1 section in G major, tempo ≈72 BPM, duration ≈13 s, all assertions PASS.

Key design decisions faithful to the documents
Concert pitch only — score.toSoundingPitch() called before any note extraction; transposition never appears in NoteObject
Flat collections — all notes/sections/spans/tempos stored as flat sorted lists
Hybrid coordinates — every NoteObject carries both bar/beat/subdivision and time_onset/time_duration
Shape variant pre-pass — variant_index assigned deterministically (round-robin seeded at 42) at parse time
CollisionCluster empty at parse time — per spec; populated by Collision Resolver in Phase 2+
No color in the data model — only instrument_family and circle_of_fifths_pos stored; UI maps to color

Quick setup (Windows):

Download Python 3.11+ from python.org — check "Add Python to PATH" during install
Open a new terminal in the project folder and run:

pip install -r requirements.txt
uvicorn backend.main:app --reload --port 8000
In a second terminal:

python tests/test_parse.py
Alternatively, if you use conda/miniconda, create a fresh environment:


conda create -n symphony python=3.11
conda activate symphony
pip install -r requirements.txt
uvicorn backend.main:app --reload --port 8000
